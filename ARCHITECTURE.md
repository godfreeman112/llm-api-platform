## 数据流图

### 用户登录流程

```
用户输入账号密码
       │
       ▼
  POST /api/auth/login
       │
       ▼
  验证用户名和密码
       │
       ├─ 失败 → 返回401错误
       │
       ▼ 成功
  生成JWT Token
       │
       ▼
  返回Token和用户信息
       │
       ▼
  前端保存Token到localStorage
       │
       ▼
  后续请求携带Token
```

### API调用流程

```
用户在聊天界面发送消息
       │
       ▼
  POST /api/chat/send
  (携带JWT Token)
       │
       ▼
  JWT认证中间件验证Token
       │
       ├─ 无效 → 返回401
       │
       ▼ 有效
  检查用户配额
       │
       ├─ 超额 → 返回403
       │
       ▼ 有配额
  获取模型配置
       │
       ▼
  转发请求到大模型API
       │
       ├─ 失败 → 记录错误日志
       │          返回502
       │
       ▼ 成功
  记录使用日志
       │
       ▼
  更新用户配额
       │
       ▼
  返回响应内容
```

## 技术选型说明

### 为什么选择SQLite？

**优点:**
- ✅ 零配置，无需单独安装数据库服务器
- ✅ 单文件数据库，备份简单
- ✅ 适合中小规模应用（并发<1000）
- ✅ ACID兼容性好
- ✅ 跨平台

**局限性:**
- ❌ 高并发性能不如PostgreSQL/MySQL
- ❌ 不支持水平扩展

**适用场景:**
- 企业内部系统（用户数<500）
- 日API调用量<10万次
- 单机部署即可满足需求

### 为什么选择Vue3 + Element Plus？

- **Vue3**: 性能好，生态成熟，学习曲线平缓
- **Element Plus**: 企业级UI组件库，开箱即用
- **Pinia**: Vue3官方推荐的状态管理方案
- **ECharts**: 强大的数据可视化库

### 为什么选择Node.js + Express？

- **轻量级**: 相比Java Spring Boot更轻量
- **快速开发**: JavaScript全栈，开发效率高
- **丰富的生态**: npm包丰富
- **易于维护**: 代码简洁，易于理解

## 性能优化建议

### 前端优化

1. **代码分割**: Vite自动进行code splitting
2. **懒加载**: 路由级别组件懒加载
3. **CDN加速**: 生产环境可使用CDN加载Element Plus
4. **Gzip压缩**: Nginx启用gzip

### 后端优化

1. **连接池**: SQLite本身是文件锁，不需要连接池
2. **缓存**: 可添加Redis缓存热点数据
3. **异步处理**: 日志记录异步化
4. **限流**: 使用express-rate-limit防止滥用

### 数据库优化

1. **索引**: 已为常用查询字段创建索引
2. **分页**: 大数据量查询使用分页
3. **定期清理**: 定期清理旧的使用日志

## 扩展性设计

### 水平扩展方案

如果需要支持更多用户和更高并发：

1. **数据库迁移**: SQLite → PostgreSQL
2. **会话存储**: 使用Redis存储session
3. **负载均衡**: Nginx多实例负载
4. **容器编排**: Docker Swarm或Kubernetes

### 功能扩展点

1. **新增模型提供商**: 在模型管理中添加配置即可
2. **自定义计费规则**: 修改usage_logs的费用计算逻辑
3. ** webhook通知**: 在关键操作后触发webhook
4. **审计日志**: 扩展usage_logs记录更多操作

## 故障排查

### 常见问题

1. **服务无法启动**
   - 检查端口占用
   - 查看日志: `docker-compose logs`

2. **数据库锁定**
   - 检查是否有多个进程访问
   - 重启服务

3. **API调用失败**
   - 检查模型配置
   - 验证API密钥
   - 查看usage_logs表的错误信息

4. **内存不足**
   - 限制Docker容器内存
   - 清理旧日志数据
